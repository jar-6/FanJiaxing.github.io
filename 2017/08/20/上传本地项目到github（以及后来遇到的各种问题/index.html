上传本地项目到github（以及后来遇到的各种问题

Posted on 2017-08-20

最近在练习用vue做一个播放器，最后在上传代码时遇到了一些问题，整理一下上传步骤和注意事项。
上传步骤如下：
进入项目文件夹，touch README.md
git init。【创建一个仓库，这时文件夹里会出现一个.git文件夹】
git add README.md
git add .【注意 .点】
git commit -m “注释文字”【要加上-m “注释文字“”，否则会出现git aborting commit due to empty commit message，我是被卡在这里了。。】
git remote add origin git@地址~
git push origin master【第一次把push打成pull了，后来看到一篇文章里写如果push不成功可以先pull以下再push】

终于成功了

08.26更
今天更新项目的时候遇到的问题
push不上去，让我先pull，pull的时候进到vim里。。
解决方法：

  1. 按i 进入insert模式，修改最上面那行黄色合并信息（可以不修改）
  2. 按键盘左上角”Esc”
  3. 输入”:wq”,注意是冒号+wq,按回车键即可
    由于我当时没这么做，直接ctrl+z了，于是没pull成功，再pull，出现：swap file already exists
    解决方法：把那个swp文件删了。。
    然后再pull，又出现：You have not concluded your merge (MERGE_HEAD exists).
    解决方法：
  4. 保留你本地的修改git merge –abort，git reset –merge，合并后记得一定要提交这个本地的合并，然后在获取线上仓库git pull
    2.down下线上代码版本,抛弃本地的修改git fetch –all，git reset –hard origin/master

08.27再更
为什么总是这么戏剧，在我又要放弃的时候成功了。。今天太刺激了
想给vue项目弄一个展示的demo，一直不成功，打开页面是空白的（之前本地打开也是空白，但是我改了一下dist里的index文件，把src改了，本地就能打开了，但是github上打开还是白的），
然后我把dist文件夹拿出来设成gh-pages分支（用git subtree push –prefix=dist origin gh-pages），但是自动生成的gh-pages页面还是指向根目录的index。。。（有一次我把github生成ghpages选项改了一下，然后指向了dist文件夹的index，但是后来我再push后又变回去了而且改不回来了，不知道为啥）
后来我做了以下事情，不知道是因为什么成功的。。

  1. 我把github默认branch从master改到ghpages上了，然后push的
  2. 我把build文件夹里的webpack.prod.conf.js里的output中加上了publicPath:’/‘
    （ps之前失败可能是因为dist有一部分没push上去，因为不小心放到ignore里了）
  3. 我在dist中npm install了一下http-server（全局和非全局都来了一下。。。）
    然后我绝望的再次push，就成功了！~
    （ps电脑中这个不是master分支，是gh-pages分支，之前一直无法push，后来我是用的git push origin gh-pages:master -f
