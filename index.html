多站点配置

  1. 添加站点

    deny from all（拒绝其他的外部访问） //deny改成allow

  4. 重启所有服务
  5. 建立网站代码
    在根目录下 创建多个站点的文件夹，分别新建PHP
  6. 在host文件中添加站点
    打开c:/windows/system32/drivers/etc/hosts 添加站点127.0.0.1 域名1&2..
    指向本地localhost 如果碰到test01.com域名时 优先向127.0.0.1请求资源

一、执行环境及作用域（p73）

  • 执行环境：
    执行环境定义了变量或函数有权访问的其他数据，每个执行环境都有一个变量对象。
    全局执行环境被认为是window对象，索引全局变量和函数都是window对象的属性和方法
  • 作用域链：
    顺序：当前执行代码所在环境的变量对象——外部环境——外部环境——全局执行环境
  • 延迟作用域链：
  1. try-catch语句中catch块
  2. with语句
    -没有块级作用域（所以变量弄不好就会乱）
  3. 初始化变量没有var声明，会自动添加到全局环境
  4. 用变量时会沿着作用域链搜寻,所以会被覆盖（如果变量被定义多次会用作用域链前端的那个。)

二、对象的原型&继承


  • 读取属性时会沿着原型链搜寻,所以会被覆盖（如果有多个属性会用实例的那个。）
  • 相关方法：
  1. ~.hasOwnProperty(“property”)：检测属性是否存在,在实例中返回true
  2. “property” in ~：检测属性是否存在,在实例和原型中都返回true
    3.~.keys():取得所有可枚举属性
    4.~ instanceof 原型：测试实例和原型链中出现的构造函数

创建对象方法

  • 创建一个Object实例，添加属性方法。var xx=new Object();xx.oo=”~”,xx.oo2=function(){}
  • 字面量语法：var xx={xx:”~”,xx2:function(){}}
  • 工厂模式：适用于批量
    把在函数中创建实例，参数传入函数：
    function createXx(oo1,oo2..){var xx=new Object; xx.oo1=oo1; xx.oo2=function(){?oo2};return xx}
    var xx实例=createXx(oo1,oo2..);
  • 构造函数模式：解决了对象识别问题
    把前者的函数名换成自定义构造函数、函数中不new Object&不return、函数后var实例时var的是自定义的对象。
    function Xx(oo1,oo2..){xx.oo1=oo1; xx.oo2=function(){?oo2};}
    var xx实例=Xx(oo1,oo2..);
  • 原型模式：所有对象共享属性方法
    把前者的参数去掉，添加属性时加上prototype
    function Xx(){xx.prototype.oo1=oo1; xx.prototype.oo2=function(){?oo2};}
    var xx实例=Xx();
  • 把前者函数中的所有内容用字面量表示：比较好写
    但是相当于重写了prototype，导致
  1. constructior属性不再指向自定义原型，所以要加上’constructor:自定义原型’这句，但是后加上的是可枚举的
  2. 切断原型和实例的关系，实例不可调用原型中的方法
  • 组合使用构造函数模式和原型模式！！！！
    构造函数模式定义实例属性，原型模式定义方法和共享属性
    function Xx(oo1,oo2..){xx.oo1=oo1; }
    Xx.prototype={constructor:Xx, oo2:function(){?oo2};}

    组合继承

    用原型链继承原型属性和方法，用构造函数继承实例属性
    function SubXx(oo1,oo2,oo3){
    Xx.call(this,oo1); //用原型链继承属性
    this.oo3=oo3 //用原型链新建属性
    }
    SubXx.prototype=new Xx(); //用构造函数继承方法
    SubXx.prototype.constructor=SubXx; SubXx.prototype.oo4 //用构造函数新建方法

一、安装

首先安装了node.js、npm、webpack、vue-cli等

这都是啥：

  • node.js：是一个Javascript运行环境(runtime)。 简单的说就是运行在服务端的 JavaScript。使用事件驱动， 非阻塞I/O 模型而得以轻量和高效，非常适合在分布式设备上运行数据密集型的实时应用。
  • npm：node.js下的包管理器。大家都把自己开发的模块打包后放到npm官网上，如果要使用，直接通过npm安装就可以直接用，不用管代码存在哪，应该从哪下载。包之间可能相互依赖，npm可以根据依赖关系，把所有依赖的包都下载下来并管理起来。
  • webpack：将vue组件（vue一个组件把三层整合到一起）翻译和打包成 .js文件
  • vue-cli：用来生成模板
  • vuex：将状态管理单独拎出来，集中式管理组件状态（如组件显示/隐藏）&组件间的数据通信，方便维护修改。用单向数据流的方式，用户界面负责触发动作（Action)进而改变对应状态（State），从而反映到视图（View）上
  • axios/vue-resource：基于 promise 的 HTTP 库
    （另外，在github上找了一些别人做的项目用来学习，发现cnpm install 总报错，于是又安装了）
  • sass（要装sass需要装ruby，于是又装了ruby。。。但是最后我还没有用到）

上传本地项目到github（以及后来遇到的各种问题

Posted on 2017-08-20

最近在练习用vue做一个播放器，最后在上传代码时遇到了一些问题，整理一下上传步骤和注意事项。
上传步骤如下：
进入项目文件夹，touch README.md
git init。【创建一个仓库，这时文件夹里会出现一个.git文件夹】
git add README.md
git add .【注意 .点】
git commit -m “注释文字”【要加上-m “注释文字“”，否则会出现git aborting commit due to empty commit message，我是被卡在这里了。。】
git remote add origin git@地址~
git push origin master【第一次把push打成pull了，后来看到一篇文章里写如果push不成功可以先pull以下再push】

终于成功了

  3. 输入”:wq”,注意是冒号+wq,按回车键即可

    解决方法：把那个swp文件删了。。
    然后再pull，又出现：You have not concluded your merge (MERGE_HEAD exists).
    解决方法：
  4. 保留你本地的修改git merge –abort，git reset –merge，合并后记得一定要提交这个本地的合并，然后在获取线上仓库git pull
